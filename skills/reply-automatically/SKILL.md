# Draft Reply

与えられたコンテキスト（アクティブウィンドウの内容・貼り付けられたテキスト・会話履歴など）を読み、文脈を自動判定して最適な返信文を生成する。**出力は返信文のみとする。**

## インプット

- 返信対象となるコンテキストが渡される。典型的には以下のいずれか。
  - アクティブウィンドウの内容（メール本文、Slack、チャット画面など）
  - 貼り付けられたテキスト
  - 直前の会話履歴
- コンテキストが返信対象として不十分・空の場合は、返信文を作らず「返信対象の内容が見つかりません」とだけ返す。

## 判定ロジック

返信文を書く前に、コンテキストから以下の4軸を判定する。判定は内部で行い、**判定結果そのものは出力しない**。

### 1. 言語
- 相手が使っている言語を特定し、返信も同じ言語で書く。
- 複数言語が混在する場合は、相手の主たる本文の言語に合わせる。

### 2. 媒体
- メール／Slack／チャット／その他 を判定する。
- メール：件名を除いた本文のみを書く。改行・段落で構成し、宛名と締めを含める。
- Slack・チャット：簡潔に。過剰な挨拶や署名は付けない。

### 3. 相手
- 社外／社内 を判定する。
- 社外：丁寧な敬語。初対面と既知の関係を文面から推定し、トーンを調整する。
- 社内：簡潔・要点中心。過度な敬語は避ける。

### 4. 目的
- 回答／依頼への返答／確認／その他 を判定する。
- 相手が求めているアクションに直接応える内容にする。曖昧に終わらせない。

## 出力仕様

- **返信文のみを出力する。**
- 判定結果・解説・前置き（「これはビジネスメールですね」等）・後書きは一切出力しない。
- コードブロックで囲まず、そのまま貼り付けられるプレーンテキストとして出力する。
- 確定していない数値・事実・約束は含めない。埋めるべき箇所がある場合は `[ ]` のプレースホルダを置く。
- 「〜について。」で終わる体言止めの文は使わない。文末は具体的な動詞・内容で締める。

## 例（抜粋）

より多くの例は `references/examples.md` を参照（Raycast は SKILL.md のみ読み込むため、代表例をここにも掲載）。

**社外・メール・日本語（依頼への返答）**
入力：「先日の件、社内で検討したいので資料をご送付いただけますか。」
```
田中さま

お世話になっております。ご検討いただき、誠にありがとうございます。

ご提案資料を添付いたしますので、ご高覧いただけますと幸いです。ご不明な点がございましたらお気軽にお申し付けください。

ご検討のほど、何卒よろしくお願いいたします。
```

**社内・Slack・日本語（確認）**
入力：「明日の定例10時開始で合ってる？アジェンダ共有しておいて。」
```
10時開始で合ってます。アジェンダは今日中に共有します。
```

## 導入（Raycast）

- スキルフォルダに `draft-reply/` ごと配置する。デフォルトの走査先は `~/.config/raycast/skills`（他AIと共用するなら `~/.claude/skills` や `~/.config/agents/skills` でも可）。
- フォルダ名はフロントマターの `name`（`draft-reply`）と一致させる。ファイル名は `SKILL.md`（大文字・厳密）。
- AI Chat / Quick AI で利用可能（AI Command ではスキルは使われない）。自動検出がオフの場合は `@draft-reply` で明示呼び出し。
